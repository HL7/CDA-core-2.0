RoleLinkType    

A code specifying the meaning and purpose of every RoleLink instance. Each of its values implies specific constraints to what kinds of Role objects can be related and in which way.

This table controls values for structural elements of the HL7 Reference Information Model. Therefore, it is part of the Normative Ballot for the RIM.

Lvl Type, Domain name and/or Mnemonic code Concept ID Mnemonic Print Name Definition/Description
1 S: RelatedLinkType (REL) V19615 REL related

The target role is related in some unspecified manner to the source role.

Discussion:no This typeCode should *only* be used when there are more-specific semantics to the relationship other than "related in some way". More specific semantics should use one of the specializations of this concept, proposing a new specialization If necessary.

2   L:  (BACKUP) 20852 BACKUP is backup for

This relationship indicates the source Role is available to the target Role as a backup.

Open Issue: PM TC needs to add an example that includes the notion of "covers for", "supports," "delegates." Need to consider additional concepts and a more precise definition.

2   L:  (DIRAUTH) 11605 DIRAUTH has direct authority over

The source Role has direct authority over the target role in a chain of authority.
2   L:  (INDAUTH) 11606 INDAUTH has indirect authority over

The source Role has indirect authority over the target role in a chain of authority.
2   L:  (PART) 11604 PART has part

The target Role is part of the Source Role.
2   L:  (REPL) 15952 REPL replaces

This relationship indicates that the source Role replaces (or subsumes) the target Role. Allows for new identifiers and/or new effective time for a registry entry or a certification, etc.
